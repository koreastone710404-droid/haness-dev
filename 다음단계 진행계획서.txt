1. 현재 상태 (1단계 완료)
────────────────────────────────────────────────
- 2열 레이아웃 (입력 패널 / 3D 씬)
- 층별 stepback 박스 기반 건축 매스
- 정북일조권 제한면 오버레이 (법정 10m / 서울 9m 선택)
- N 화살표, 치수 라벨, 격자, 토글 컨트롤
- 플로팅 요약 패널 및 면책 배너
- 용도지역 프리셋 21종 (국토계획법 시행령 §84·§85 법정 최대치 반영)
- 기술 스택: React + Vite + three.js (@react-three/fiber, drei)

────────────────────────────────────────────────
2. 2단계 구성 요소 (권장 진행 순서)
────────────────────────────────────────────────

[1] 지자체 조례 · 지구단위계획 기반 규제 정밀화
  - 외부 의존성: 주요 지자체 조례 수집
  - 예상 규모: 1~2주
  - 내용:
    * 법정 최대치가 아닌 지자체별 실제 적용 상한(예: 서울시 2종일반 200%)
    * 지구단위계획·재정비촉진지구 완화 규정
    * 방화지구 내화구조 건폐율 완화
    * 사용자 수동 덮어쓰기 + 프리셋 DB 매칭

[2] 태양 궤적 기반 그림자 시뮬레이션
  - 외부 의존성: 없음 (위도 · 일시 입력으로 계산)
  - 예상 규모: 5~7일
  - 내용:
    * 위도 · 경도 · 일시(월/시각) 입력 UI
    * 태양 고도 · 방위각 계산 (SunCalc 라이브러리 또는 자체 구현)
    * 동하지 / 춘추분 기준 그림자 궤적 시각화
    * 시간 슬라이더로 그림자 이동 애니메이션
    * 사업성 검토·일조권 검증 양쪽 모두에 유용

[3] 가로구역별 최고높이 DB
  - 외부 의존성: 지자체 공개자료 수집 필요
  - 예상 규모: 1~2주
  - 내용:
    * 주요 지자체(서울 · 부산 · 경기 주요시) 가로구역별 최고높이 자료 수집
    * 주소 / 지번 기반 매핑 룩업 테이블 구축
    * 자동 적용 + 수동 덮어쓰기 옵션

[4] VWorld GIS API 연동 (지번 → 실제 대지 경계선)
  - 외부 의존성: VWorld API 키 발급 필수, 백엔드 프록시 권장
  - 예상 규모: 2~3주
  - 내용:
    * VWorld 개발자센터 API 키 발급
    * CORS 대응용 백엔드(Node.js 등) 프록시 구성
    * 지번 입력 시 대지 폴리곤 조회 · 렌더링
    * 직사각형 가정을 실제 다각형 지적선으로 대체
    * 좌표계(EPSG:5186 / WGS84) 변환 처리

3. 2단계에서 제외한 항목: 도로사선 제한
────────────────────────────────────────────────
※ 구 건축법 §60(도로사선 제한)은 **2015년 5월 18일 개정으로 폐지**되어
  일반 대지에는 더 이상 적용되지 않음.

다만 다음 경우에는 여전히 높이 제한이 부과될 수 있어 별도 기능으로
검토 가능:
  - 지구단위계획구역에서 개별 규정한 높이/사선
  - 고도지구 지정 구역
  - 재정비촉진지구 특별 규정
  → [3] 가로구역별 최고높이 DB 기능에 통합 검토

4. 권장 시작점
────────────────────────────────────────────────
→ [2] 태양 궤적 기반 그림자 시뮬레이션부터 착수
  - 사유:
    1) 외부 API · 데이터 수집 없이 즉시 개발 가능
    2) 정북일조권 규제의 실제 효과를 시각적으로 검증 가능
       (이미 구현된 정북일조권 제한면과 상호 보완)
    3) 사업주 · 토지 소유자에게 가장 직관적인 가치 제공
    4) 순수 기하 계산이라 테스트 · 유지보수가 용이

차순위 추천: [1] 지자체 조례 DB
  - 현재 "법정 최대치"를 표시하지만 실무에서는 지자체별로 더 낮은 값이
    적용됨. 서울시 조례(예: 2종일반 200%) 같은 대표 지자체 프리셋부터 추가.
