# Pull Request Template

## 제목

- 변경 내용을 간결하게 요약합니다. 예: `feat: 로그인 UI 개선`, `fix: API 인증 오류 해결`

## PR 요약

- 변경 목적과 주요 내용을 간단히 정리합니다.

## 변경 사항

- 무엇을 변경했는지 항목별로 작성합니다.
- 새로운 기능, 수정한 버그, 제거한 코드, 추가한 의존성 등을 포함합니다.

## 검증 방법

- 변경 사항이 제대로 동작하는지 확인한 방법을 명시합니다.
- 테스트 명령어, 수동 검증 절차, 확인한 결과를 적습니다.
  - 예: `npm run test`, `pytest tests/test_user.py`, 브라우저에서 로그인 확인

## 체크리스트

### 기본
- [ ] 코드가 빌드되고 실행되는지 확인했습니다. (`npm run build`)
- [ ] 관련 테스트를 작성하거나 업데이트했습니다. (`node tests/envelope.test.mjs`)
- [ ] UI 변경 시 런타임 테스트를 통과했습니다. (`node tests/runtime.test.mjs`)
- [ ] 린트/포맷 도구를 실행했습니다.
- [ ] 문서나 주석이 필요한 경우 함께 업데이트했습니다.
- [ ] 관련 이슈를 연결했습니다.

## 관련 이슈

- 이슈 번호 또는 링크를 작성합니다. 예: `#123`

## 참고 사항

- 배포 시 유의할 점, 롤백 계획, 임시로 남긴 TODO 등을 작성합니다.
- 리뷰어가 알아야 할 추가 정보가 있다면 여기에 작성합니다.
