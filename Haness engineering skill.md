## 개요

`Haness Engineering` skill은 한스 엔지니어링 팀의 개발자 및 협업자를 위해 설계된 맞춤형 가이드라인 문서입니다. 이 스킬은 코드 작성, 리뷰, 테스트, 배포, 문서화, 기술 결정 등 소프트웨어 엔지니어링 전반에서 일관되고 효율적인 결과를 내기 위한 지침을 제공합니다.

## 목적

- 프로젝트 품질 향상
- 일관된 개발 표준 유지
- 빠른 문제 해결과 명확한 커뮤니케이션 지원
- 팀 내 지식 공유와 협업 생산성 증대

## 주요 원칙

1. **명확성 우선**
   - 코드와 문서는 읽기 쉬워야 합니다.
   - 변수, 함수, 파일 이름은 역할을 명확히 표현해야 합니다.

2. **간결성 유지**
   - 불필요한 복잡성을 피하고, 단순한 해결책을 선호합니다.
   - 반복되는 코드는 추상화하거나 재사용 가능한 방식으로 정리합니다.

3. **테스트 중심 개발**
   - 새로운 기능과 버그 수정은 가능한 한 자동화된 테스트로 검증합니다.
   - 테스트는 기능 요구사항을 명확히 반영해야 합니다.

4. **안정성 확보**
   - 변경 사항은 안전하게 통합되고, 위험 요소는 적절히 관리합니다.
   - 린트, 형식 검사, 정적 분석 도구를 활용합니다.

5. **협업과 문서화**
   - 코드 리뷰와 PR 설명을 통해 지식을 공유합니다.
   - 주요 아키텍처 결정과 운영 절차는 문서로 남깁니다.

## 사용 시나리오

- **기능 개발**: 요구사항을 명확하게 파악하고, 설계와 구현을 일관되게 유지합니다.
- **버그 수정**: 원인 분석, 간단한 재현 절차, 수정 범위 최소화 원칙을 따릅니다.
- **코드 리뷰**: 기능 의도, 구현 정확성, 성능, 보안, 유지보수성 관점에서 검토합니다.
- **문서 작성**: API, 배포, 개발 환경, 테스트 방법을 포함한 실용적인 문서를 작성합니다.

## 실행 지침

- 변경 전후의 차이를 명확히 설명합니다.
- 코드 스타일과 프로젝트 컨벤션을 준수합니다.
- 팀의 기존 솔루션과 아키텍처를 먼저 검토합니다.
- 필요 시 기술 부채와 리팩토링 가능성을 정리합니다.

## 기대 효과

- 일관된 코드 베이스 유지
- 유지보수 비용 절감
- 새로운 팀원 온보딩 속도 향상
- 더 빠르고 신뢰성 높은 개발 프로세스

## 참고

이 문서는 `Haness Engineering` 팀이 소프트웨어 개발 활동에서 정리된 기준과 목표를 공유하는 데 도움이 되도록 설계되었습니다. 실제 프로젝트에서는 팀 내 규칙과 환경에 맞게 추가 수정 및 확장을 권장합니다.

## 에이전트 환경과의 관계

이 skill은 **사람 개발자용 가이드라인**이고, [SKILL.md](SKILL.md)의 하네스 엔지니어링은 **AI 에이전트용 환경 설계 방법론**입니다. 두 문서는 서로 보완합니다.

- 같은 규칙을 양쪽에 중복해 적는 대신, 공통 규칙은 기계적으로 강제 가능한 형태(린트·테스트)로 옮기고 [AGENTS.md](AGENTS.md)의 **불변식(Invariants)** 섹션에 요약한다.
- 새 팀 표준이 생기면: (1) 이 문서에 배경·원리를 기록, (2) 가능하면 [AGENTS.md](AGENTS.md)에 코드로 강제하는 규칙으로 반영, (3) 테스트·린트·pre-commit 훅에 추가한다.
